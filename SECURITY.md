# Инструкции по безопасности

## Первый запуск и создание администратора

При первом запуске приложения необходимо создать первого администратора.

### Способ 1: Через переменные окружения (рекомендуется)

Создайте файл `.env` в корне проекта (не включается в архив) со следующим содержимым:

```
ADMIN_USERNAME=admin
ADMIN_PASSWORD=your_secure_password_here
ADMIN_EMAIL=admin@example.com
```

**ВАЖНО:** 
- Файл `.env` должен быть добавлен в `.gitignore`
- Не передавайте файл `.env` третьим лицам
- Используйте надежный пароль

### Способ 2: Через интерфейс администратора

Если первый администратор уже создан, можно создать дополнительных пользователей через интерфейс администратора.

## Исключенные из архива файлы

Следующие файлы и папки **НЕ включены** в архив для безопасности:

- `*.db` - базы данных с пользователями и паролями
- `*.log` - файлы логов
- `__pycache__/` - кэш Python
- `venv/` - виртуальное окружение
- `.env*` - файлы с переменными окружения
- `.streamlit/` - конфигурация Streamlit
- `*.sync-conflict-*` - файлы конфликтов синхронизации
- Временные файлы

## Рекомендации по безопасности

1. **Пароли**: Используйте надежные пароли для всех пользователей
2. **База данных**: Файл `users.db` содержит хешированные пароли - храните его в безопасном месте
3. **Переменные окружения**: Используйте `.env` файлы для хранения секретов (не коммитьте их в git)
4. **Хеширование**: В текущей версии используется SHA256 для хеширования паролей. Для продакшена рекомендуется использовать bcrypt или argon2
